The crisis nobody is solving

AI coding agents have democratized software creation. But they've also created a massive gap: the code ships, but nobody checked if it should. The entire engineering discipline — architecture, security review, compliance, testing, deployment strategy, cost analysis — gets skipped when a non-technical user prompts an AI to "build me an app."

The 12 critical gaps vibe coders face

Security

Insecure code by default

Hardcoded credentials, SQL injection, weak auth, insecure deserialization — AI generates code that "works" but is exploitable. CurXecute (CVE-2025-54135) let attackers execute arbitrary commands via Cursor.

Compliance

Missing regulatory depth

GDPR, HIPAA, SOX, PCI-DSS — AI has no concept of your industry's legal requirements. A health app may log user data without encryption. A finance app may ignore rounding regulations.

Architecture

No structural planning

Without specs, agents guess at hundreds of unstated requirements. Assignment systems without permissions, APIs without rate limiting, databases without indexing.

Dependencies

Supply chain attacks

Agents install packages without version pinning or vulnerability scanning. Typosquatting attacks (e.g., "bitensor" vs "bittensor") inject malicious code automatically.

Operations

No deployment knowledge

Users have zero understanding of hosting, DNS, SSL, edge costs, CDN configuration, scaling thresholds, or infrastructure-as-code. The "build" button works but production doesn't.

Cost

Hidden infrastructure costs

Token spending spirals out of control. Cloud resources are provisioned without cost awareness. A Replit user saw their app's database deleted during AI's "optimization."

Review

Unreadable PRs

AI generates massive pull requests that no human can meaningfully review. Changes cascade across files without clear intent. Version control is an afterthought.

Testing

Zero test coverage

Speed is prioritized over safety. SAST, DAST, penetration testing, and security scanning are completely absent from vibe coding workflows.

Secrets

API key leakage

Keys hardcoded in source, pushed to public repos, leaked via MCP servers. Claude Code itself had CVE-2026-21852 allowing API key exfiltration through malicious repos.

Drift

Architectural drift

Different prompts from different sessions produce inconsistent patterns. Over time, the codebase becomes a patchwork of conflicting styles, frameworks, and approaches.

Context

Context loss between sessions

Agents forget project history, design decisions, and constraints. Every new session starts from scratch. The Cursor "code reversion bug" silently undid user changes in March 2026.

Access

CLI & tooling barrier

Claude Code is a CLI tool. Git, npm, Docker, CI/CD — these are foreign concepts to the vibe coder. The gap between "app works in preview" and "app is production-ready" is enormous.

"The problem isn't the coding agent's coding ability, but our approach. We treat coding agents like search engines when we should be treating them more like literal-minded pair programmers."

— GitHub Spec Kit team, Sept 2025

Research foundations

Our product concept is grounded in recent academic research, industry reports, and empirical studies from 2025-2026.

Key papers & findings

Safiron: Foundational Guardrail for Agentic Systems (Oct 2025)

Huang et al. propose pre-execution guardrails — intervening at the planning stage before any action is executed. Their AuraGen pipeline synthesizes risky trajectories and trains a compact guardian model (Safiron) that flags risky agent plans, assigns risk types, and generates rationales. Key insight: existing guardrails mostly operate post-execution, which is too late.

Policy as Prompt: Automated Guardrail Synthesis (Sep 2025)

A framework that translates unstructured design documents into verifiable, real-time guardrails using the "principle of least privilege." Constructs a policy tree from technical artifacts, then compiles it into lightweight prompt-based classifiers that audit agent behavior at runtime. Bridges the policy-to-practice gap.

Professional Developers Don't Vibe, They Control (Dec 2025)

Carnegie Mellon observational study of 13 professional developers. Findings: pros plan before implementing and validate all agentic outputs. Agents are suitable for well-described, straightforward tasks but not complex ones. The gap: non-professionals skip planning entirely.

Databricks Security Vibe Check (2025)

Tested Claude 3.7 Sonnet and GPT-4o on PurpleLlama's Secure Coding Benchmark. Three prompting strategies tested: security-focused system prompt, self-reflection, and iterative review. Self-reflection was most effective at reducing insecure code. This validates our core architecture — an autonomous reflection layer.

Spec-Driven Development — Thoughtworks Analysis (Dec 2025)

SDD separates design from implementation: idea → spec → plan → tasks → code. Key insight from GitHub: "We're moving from 'code is the source of truth' to 'intent is the source of truth.'" Specs are versioned Markdown that drive agent execution. But current SDD tools lack security, compliance, and novice accessibility.

AgentHelm is the autonomous pre-execution intelligence layer that sits between a user's intent and any AI coding agent. It transforms vague ideas into complete, secure, production-ready specifications — then continuously validates, monitors, and improves the agent's output. Think of it as the full engineering team that non-technical builders never had.

What AgentHelm is NOT

  • Not a code generator — it directs agents, not replaces them
  • Not a markdown/docs generator — it produces executable specifications with enforcement
  • Not a todo list — it's an autonomous system that learns, reflects, and improves
  • Not enterprise-only — built for the novice vibe coder first, scales to teams

The 7 pillars of AgentHelm

01 — PATHFINDER

Intent extraction & specification

Conversational interview that transforms vague ideas ("build me an Airbnb clone") into complete technical specifications covering architecture, data models, security requirements, compliance needs, deployment strategy, and cost projections. Industry-aware: knows healthcare needs HIPAA, finance needs SOX, EU needs GDPR.

02 — GUARDRAIL

Pre-execution security layer

Before any agent writes a line of code, AgentHelm generates security policies: input validation rules, auth requirements, secret management protocols, dependency allow-lists, OWASP compliance checks. Policies are compiled into enforceable guardrails that wrap the agent's execution.

03 — ORCHESTRATOR

Agent-agnostic execution

Works with any coding agent: Claude Code, Codex, Cursor, Replit, Bolt, Lovable, local LLM agents, OpenCode, Kimi, or custom setups. Generates agent-specific instruction files (CLAUDE.md, .cursorrules, AGENTS.md) and task breakdowns. Plug into any workflow.

04 — SENTINEL

Continuous validation loop

After each agent execution cycle, AgentHelm reviews the output: security scan, compliance check, architectural consistency, test coverage verification, dependency audit, secret leak detection. Flags issues before they hit production. Self-reflection reduces insecure code by up to 30% (Databricks).

05 — MEMORY

Persistent project intelligence

Maintains a knowledge graph of your project: design decisions, architectural patterns, constraint history, past vulnerabilities found, team preferences. Survives across sessions, across agents, across models. Prevents the "context loss" problem that plagues every current tool.

06 — DEPLOYER

Production readiness engine

Generates deployment configurations, CI/CD pipelines, monitoring dashboards, cost estimates, scaling rules, backup strategies, and incident response plans. Handles the "last mile" that kills most vibe-coded projects: DNS, SSL, environment variables, database migrations.

07 — LEARNER

Autonomous improvement system

Reinforcement learning from user feedback, agent performance metrics, and security outcomes. Gets better at predicting what your project needs. Benchmarks itself against industry standards. Simulates attack vectors. Adapts to new frameworks, new regulations, new threat models.

Delivery modes

MODE 1

CLI companion

Runs alongside Claude Code, Codex, OpenCode. Generates CLAUDE.md, watches file changes, triggers validation on each commit. helm init → helm watch → helm deploy

MODE 2

MCP server

Model Context Protocol integration. Any MCP-compatible tool (Cursor, Claude, VS Code) can connect to AgentHelm as a tool server. Agents call AgentHelm functions directly: helm.validate(), helm.spec()

MODE 3

Web dashboard

Visual interface for non-technical users. Guided interview flow, real-time security dashboard, one-click deployment. Works with cloud tools: Bolt, Lovable, Replit, AI Studio.

MODE 4

Skills / rules files

Exportable configuration files: .cursorrules, CLAUDE.md, AGENTS.md, .github/workflows. AgentHelm generates these from your spec and they work even without AgentHelm running.

Execution roadmap

Phase 1 — Weeks 1-4 (MVP)

Pathfinder CLI + Spec Engine

Build the core: conversational intent extraction → specification generation → agent instruction file export (CLAUDE.md, .cursorrules, AGENTS.md). Target: Claude Code + Cursor users. Single-industry focus: web apps. Ship as open-source CLI.

Deliverables: helm init command, spec template engine, security policy generator, 5 industry compliance templates (web, healthcare, finance, e-commerce, SaaS).

Phase 2 — Weeks 5-8 (Sentinel)

Validation loop + MCP server

Add the post-execution validation layer: file watcher that triggers security scans, compliance checks, and architectural consistency reviews after each agent commit. Ship MCP server integration so any MCP-compatible tool can call AgentHelm.

Deliverables: helm watch command, SAST integration (Semgrep), dependency scanner, MCP tool server, validation dashboard.

Phase 3 — Weeks 9-12 (Memory + Deploy)

Persistent memory + deployment engine

Project knowledge graph that persists across sessions. Deployment config generator: Docker, Vercel, AWS, Railway, Fly.io. Cost estimator. CI/CD pipeline generator. SSL/DNS guidance.

Deliverables: Vector store for project memory, deploy config templates, cost calculator, monitoring setup generator.

Phase 4 — Weeks 13-20 (Web + Learning)

Web dashboard + autonomous learning

Visual interface for non-technical users. Guided interview flow. Real-time security dashboard. Feedback-driven improvement system. Benchmark suite to demonstrate AgentHelm's impact on code quality.

Deliverables: Web app, user onboarding flow, learning system, public benchmark results, integration with Bolt/Lovable/Replit.

Phase 5 — Weeks 21-30 (Scale)

All industries + team features + marketplace

Expand compliance templates to all BLS categories. Team collaboration features. Marketplace for community-contributed spec templates, security policies, and industry packs. Enterprise tier with SSO, audit logs, custom policies.

Deliverables: 50+ industry templates, team dashboard, marketplace, enterprise features, API for third-party integrations.
